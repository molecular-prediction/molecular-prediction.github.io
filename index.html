🌍 研究背景

我们为什么要做这件事？

一个关于化学合成、逆向推理与 AI 辅助的故事

🧩 化学合成中的"逆向推理"难题

在有机合成化学中，化学家经常面临一个经典困境：已知目标分子的结构，如何找到合成它的前体？ 这被称为逆向合成分析 (Retrosynthetic Analysis)—— 从产物出发，反推所有可能的反应路径和起始原料。对于石墨烯纳米带 (GNR) 这类复杂多环芳烃， 可能的切割方式成百上千，手动穷举极其耗时且容易遗漏。

🧩

一个类比：乐高逆向拆解

想象你面前有一个拼好的乐高城堡（目标分子），你需要找出所有可能的把它拆成两个小组件（单体前驱体）的切法。 不同的切法会产生不同形状的"半个城堡"，而每种"半个城堡"对应一种不同的合成路线—— 有些容易拼接（合成），有些根本拼不上（不合法）。我们的工作就是教会计算机自动穷举所有切法，并告诉你哪种最可行。

⚠️ 核心挑战：从产物反推前体，为什么这么难？

石墨烯纳米带 (GNR) 的实验室合成遵循"自下而上" (bottom-up) 策略： 先合成"单体" (Monomer) 前驱体小分子，再通过 Ullmann 偶联和Scholl 环化脱氢 反应聚合成最终的纳米带。这个过程的逆向推演面临三大困难：

① 切割方案海量：一个含有 N 个苯环的分子，理论上存在指数级数量的切割路径， 且需要考虑周期性边界条件（分子在水平方向无限重复）；
② 化学合法性约束：并非所有切割方案都能产生化学稳定的前体—— 切割位置必须避开边缘苯环、封端基团（Br、甲基）必须精准配对；
③ 合成难易度评估：即使生成了一批候选前体，如何判断哪个最容易合成？ 这需要综合考量空间位阻、电子效应、反应选择性等复杂因素。

🎯 我们的目标

我们希望开发一个全自动逆向推理引擎： 输入目标纳米带的化学式（SMILES）， 输出所有可能的合成前体分子—— 同时附上每个候选前体的合成难易度评估（由 LLM 大模型给出）， 帮助化学家从数十个候选中快速锁定最优合成路线。 整个过程无需任何训练数据，纯粹基于图论算法和化学规则。

🔷

分子 = 图

每个苯环是一个节点，相邻苯环之间是边。 整个多环芳烃分子变成一个蜂窝状的平面图（Planar Graph）， 逆向切割问题转化为在图上穷举所有从上到下的合法路径。

🔄

穷举 = 不停搜索

目标分子在水平方向是周期重复的，我们只需在一个"周期单元"内搜索所有切割路径， 然后像壁纸拼接一样复制到全局，确保不遗漏任何一种合法切割方案。

📏

化学规则 = 过滤器

并非所有切割方案都能产生稳定的前体——"边缘苯环不能横切""封端基团必须配对"…… 我们将这些化学经验转化为严格的数学约束，自动过滤掉不合法方案。

🤖

LLM = 智能排序

即使穷举出几十种候选前体，化学家也难以逐一评估。我们调用大语言模型 自动评判每个前体的合成难易度，按可行性从高到低排序，帮助化学家优先尝试最优路线。

🧭 第一步：给分子"画地图"

坐标重建 · 苯环识别 · 蜂窝网格化

当我们输入一串 SMILES 代码（例如一个含有上百个碳原子的多环芳烃）， 计算机并不知道这些原子在空间中的位置。我们使用 RDKit（化学信息学工具包） 计算出每个原子的二维坐标，然后用 SSSR 算法（最小环集搜索）识别出所有的六元环。

接着，我们按 Y 坐标聚类 将苯环分层（确定每个环在"第几排"）， 按 X 坐标排序 确定列（第几个位置），最终得到一个规整的 蜂窝网格——就像给分子画了一张精确的"地图"。 进一步，通过计算苯环中心的几何距离，建立物理邻接关系——计算机知道了每个环的"邻居是谁"。

🔍 第二步：在地图上"找路"

DFS 深度优先搜索 · 周期性边界 · 边缘约束规则

有了地图，下一步就是在图上找一条从顶到底的切割路径。我们使用 DFS（深度优先搜索）——一种经典的图遍历算法： 从顶部任意一个苯环出发，每一步沿邻居向下或斜向走，直到到达底部。

v2.0 的核心创新——周期性边界条件： 传统做法中，路径被限制在固定宽度的图块内，走到左边界就"撞墙"了。 我们引入了取模运算（target_col % k）： 当路径走向左边界时，算法自动"看到"右边界的对应节点——就像玩《贪吃蛇》游戏时， 蛇从屏幕左边出去、右边进来。这使得我们能够模拟无限延伸石墨烯上的斜向切割。

三条安全规则：在搜索过程中，我们嵌入了化学家的经验—— ① 顶层不允许横向移动；② 到底层立即停止；③ 边缘苯环不能横切。 违反任何一条规则的路径都会被自动丢弃。

🧪 第三步：给分子"戴帽子"

封端化学 · Alpha/Beta 键分类 · 异构体穷举

找到切割路径后，我们沿着路径把分子"剪开"。但剪开处会留下不饱和的化学键（悬挂键）—— 在现实中这些位置需要被特定的原子"封端"（Capping），就像给断裂的绳子打结。

智能键分类系统：我们区分两种关键的断键位置—— Alpha 键：连接两个相邻苯环"桥头碳"的键（Ullmann 偶联位点），需要加 Br（溴原子）； Beta 键：海湾位（Scholl 反应区域）的键，需要加 C（甲基）。

对于最常见的 K=2（双苯环窗口）情况，我们在左右两侧各放置一个 Br， 然后用 C（甲基）封端其余位置。由于每个位置有多个候选， 系统自动穷举 4 种甲基异构体——每种对应不同的合成路线和电子性质。

v2.0 的三大技术突破

🔄 创新一：周期性边界条件 —— 让路径"穿墙而过"

❌ v1.0 的问题

切割路径被限制在宽度为 K 的图块内部。当路径向左下方延伸触碰到左边界时， 搜索立即终止——就像一只蚂蚁在封闭的盒子里爬行，走到墙边就无路可走了。 这导致无法找到长斜向切割路径，严重限制了可生成的纳米带种类。

✅ v2.0 的解决方案

引入逻辑邻居映射：在路径搜索中执行取模运算 target_col % k_cols。 当搜索到左边界时，算法自动"看到"右边界的对应节点（反之亦然）。 就像把一张纸的两边粘起来形成圆筒——蚂蚁可以一直爬下去， 完美模拟无限延伸石墨烯上的斜向切割。

📐 创新二：三维边缘几何约束 —— 把化学经验变成数学规则

⬆️

顶层约束

切割必须从分子最顶层开始，且第一步严禁横向移动。 确保切割从顶部开始向下延伸，而非在顶部徘徊。

⬇️

底层约束

路径一旦触及最底层苯环立即终止。严禁在最底层横向移动， 保证切割干净利落地贯穿整个分子高度。

↔️

侧边双向封锁

横向移动涉及的任意一个节点是边缘苯环，该移动即被禁止。 从根本上杜绝了破坏边缘苯环化学稳定性的非法切割。

📏 创新三：动态偏移量学习 —— 让算法"入乡随俗"

❌ 传统做法

硬编码蜂窝网格的邻接偏移量（如上下左右斜）。假设所有分子的几何排列都是"标准"的， 但真实分子存在畸变和扭曲，固定偏移量会导致大量漏连或错连。

✅ 我们的做法

从数据中学习：遍历图块内已有的物理连接，按行奇偶性 分类，自动记录所有真实存在的 (Δrow, Δcol) 偏移模式。 样本不足时，系统自动回退到标准偏移量作为保底。 这种自适应策略使得算法能够处理几何形态各异的分子而无需人工调参。

❓ 常见问题

你可能会问

🤔 为什么不用机器学习，而是用图论算法？ ▼

机器学习方法需要大量标注数据，而对于新型 GNR 结构，我们往往没有任何训练样本。 图论方法基于确定性化学规则，不需要训练、零样本即可运行， 且结果的可解释性远超黑箱模型——每一步推理都可以追溯和验证。

🧪 生成的单体真的能用于实验合成吗？ ▼

我们生成的单体遵循了Ullmann 偶联 + Scholl 环化脱氢的成熟合成路线化学规则。 RDKit Sanitize 验证确保了价键饱和与价态合理，LLM 评判进一步审查了化学稳定性。 这些产物是化学上合理的合成前驱体，但最终仍需要实验验证。

⚡ 系统处理一个分子需要多长时间？ ▼

对于典型的分子（~100 个苯环），完整 Pipeline（解析 → 搜索 → 封端 → 评判） 通常在30 秒到 3 分钟内完成。DFS 搜索有 3 秒超时限制和 20 条路径上限， 在效率和完备性之间取得了平衡。

🔓 这个项目开源吗？我能自己部署吗？ ▼

完全开源！访问 GitHub 仓库， pip install -r requirements.txt 即可运行。 Web 界面通过 python app.py 启动，浏览器访问 localhost:8000 即可使用。
